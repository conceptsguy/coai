# ADR-0006: Context Linking via System Prompt Injection

**Status:** Accepted
**Date:** 2026-03-28

## Context

Coai's core feature is that connected chat nodes share context — a conversation about "market analysis" can inform a connected "product strategy" chat. The question is how to pass that context to the LLM.

Options:

1. **Raw message injection** — include full message history from connected nodes in the prompt
2. **Summary injection** — generate rolling summaries of each node, inject those into the system prompt
3. **Retrieval (RAG)** — embed messages, retrieve relevant chunks at query time

## Decision

Connected context is computed on-demand via `getConnectedContexts(nodeId)` which follows incoming edges, collects source node summaries, and injects them as labeled sections in the system prompt:

```
## Connected Chat: {sourceTitle}
{summary}
---
```

Summaries are generated by a cheap model (Haiku) after each assistant response and stored on the node. They are not cached separately or stored in a dedicated context table.

Rejected **raw message injection** because: a connected node with 50+ messages would consume most of the context window, leaving little room for the actual conversation. Summaries compress a full conversation into ~200 tokens.

Rejected **RAG** because: it adds infrastructure complexity (vector store, embedding pipeline) for a marginal relevance improvement. Summaries are good enough when the user has explicitly drawn a connection — the intent is clear, unlike open-ended retrieval.

## Tradeoffs

**Gained:** Simple implementation, predictable token costs, no additional infrastructure. Users control context flow explicitly by drawing edges.

**Accepted:** Summaries are only as fresh as the last assistant response in the source node. If the source chat changes significantly, the downstream chat won't see the update until the source generates a new summary. This is acceptable because context is a background signal, not a live feed.

**Deferred:** Summary staleness detection — we could show a visual indicator when a connected summary is outdated, or auto-regenerate on edge creation.
